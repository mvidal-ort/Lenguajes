Environment (ambiente-contexto)

El laboratorio 2 tenia un environment que era una tupla (Σ, Γ) de firma de funciones y declaraciones de variables

Ahora el lab 3 va tener una quintupla (δ, π, κ, σ, γ) con

  • δ (delta) : la informacion de los parametros de tipos (o variables de tipos) de los data types declarados

Ejemplo : 

data N = O | S <N>
data List<a> =  Nil | Cons <a, List<a>>
data Arbol<a,b> = Hoja<a> | NodoInterior<b,Arbol<a,b>,Arbol<a,b>>

En este ejemplo seria el siguiente mapeo o diccionario

δ = [N -> <>, List -> <a>, Arbol -> <a,b>]

  • π (pi) : es una lista de parametros de tipos (variables de tipo), sin variables repetidas , como por ejemplo: <> ,<a> y <a,b> guarda el contexto de las variables de tipo que tienen sentido en un momento dado

  • κ (kappa) : es la informacion de todos los constructores que haya declarados es una mapeo o diccionario que para cada constructor me devuelva la informacion relacionada a ese constructor, basicamente la lista de parametros del constructor y ademas a que tipo al que pertence el constructor.

    En el ejemplo anterior para seria el siguiente mapeo:

    κ = [ O -> ([],N)
    , S -> ([N],N)
    , Nil -> ([], List)
    , Cons -> ([a, List<a>], List)
    ]

  • σ (sigma) : es la informacion relacionada a los metodos (o funciones) es un mapeo entre nombres de metodos y les asocia los tipos de los parametros del metodo, los tipos de que retorna, y si es polimorfica la lista de parametros de tipos

    method suma (n : N, m : N) returns (r : N)
method length<a> (l : List<a>) returns (r : N, m : N)


σ = [ suma -> (<>, [N,N], [N])
    , length -> (<a>, [List<a>], [N, N])
    ]

  • γ (gamma) : es la informacion de tipo de las variables y como Imp tiene bloques de declaraciones de variables va ser una lista de mapeos o diccionarios, igual que en el laboratorio 2

Por ejemplo para chequear que Arbol<N,List<N>> es un tipo valido, vemos que esta aplicado a una cantidad de tipos igual a los parametros de tipos declarada en su de su defincion (<a,b>), en este caso 2, y luego recursivamente chequea que cada argumento sea a su vez un tipo valido.

-------------- a ∈ π
(δ , π) ⊢ a type

 (δ , π) ⊢ t < α1..αn > type    |δ(t)| = n          (          | <a> | = 1                    )
         ^     ^
         |     | ejemplo
        List < N >

Tenemos que ver que: * Cons este declarado en κ , haciendo el lookup κ (Cons) = ([α1 = a, α2 = List]], List) * βi sean "instancias" de los αi, en este caso: * β1 = N sean "instancias" de los α1 = a, es igual si cambiamos a por N, teniendo que sustitucion θ = (a -> N) * β2 = List sean "instancias" de los α2 = List, es igual si cambiamos a por N, teniendo que sustitucion θ = (a -> N) * Para ver el tipo de la expresion resutlante tengo que hacer δ(List) = , y esto le aplico la sustitucion θ que hallamos, o sea a le cambiamos a por N y me queda
